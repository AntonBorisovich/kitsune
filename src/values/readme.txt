В папке values хранятся значения в формате json, которые может использовать бот.
Каждое значение должно иметь одно и то же имя как в имени файла, так и внутри него.

Пример содержания файла stroka.json (строка):
{"stroka": "значение"}

Пример содержания файла money.json (число):
{"money": 1337}

Пример содержания файла znachenie.json (логический):
{"znachenie": true}

Пример содержания файла whitelist.json (массив):
{"whitelist": ["Дима", "Илья", "AYAYA"]}

Значения, которые может использовать бот и их описание:
 discordtoken - токен для входа в discord (строка) [обязателен]
 prefix - префикс для общения с ботом (строка) [обязателен]
 twofasecret - секрет для генерации 2FA кода (строка)
 >Используется в update, там же и генерируется. Если задать самостоятельно то команда может работать не корректно (строка)
 maintenance - режим тех. обслуживания (логический)
 >Режим тех. обслуживания разрешает общение с ботом только людям, указанным в testers.json или developers.json
 testers - список тестеров, а именно их id в дискорде (массив)
 developers - список разработчиков, а именно их id в дискорде (массив) [обязателен]
 >Первый id в массиве будет отображен в info
 debug - режим отладки (логический)
 >Каждый вызов бота будет сопровождатся полным логированием в лс людям, указанным в developers.json, а так же пометкой в каждом сообщении, что отчёт о выполненой команде был отправлен разработчикам. Общаться же с ботом могут только люди из testers.json и developers.json
 version - отображаемая версия бота (строка)
